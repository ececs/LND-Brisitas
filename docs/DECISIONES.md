# Decisiones y criterios

## Tecnicas
- HTML estatico para cada pagina por simplicidad y claridad.
- CSS global en `styles.css` para consistencia visual.
- JavaScript en `script.js` para la logica compartida.

## Funcionales
- Carrito persistente con `localStorage` para no perder el estado al recargar.
- Checkout simulado sin pasarela real (proyecto academico).
- Panel admin local para alta de productos sin backend.

## Usabilidad
- Navegacion principal consistente en todas las paginas.
- Botones y formularios con etiquetas claras.
- Mensajes de estado para acciones clave (carrito y formularios).

## Limitaciones
- No hay autenticacion real.
- No hay control de stock en servidor.
